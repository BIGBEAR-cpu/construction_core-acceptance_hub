# 开发报告模板（Development Report Template）

## 任务目标
- 简述任务目标与边界。

## 修改文件清单
- 列出本次修改的已有文件。

## 新增文件清单
- 列出本次新增文件。

## 删除文件清单
- 若无删除，填写“无”。

## 核心实现说明
- 说明关键实现思路、关键逻辑与设计取舍。

## 已运行测试
- 列出执行过的测试命令。

## 测试结果
- 说明测试通过/失败情况与证据。

## 未完成事项
- 列出尚未完成的任务点。

## 已知风险
- 列出当前已知风险与影响范围。

## 需要人工验证的事项
- 列出必须人工验证的环节与建议操作。

## 建议提交信息
- 提供建议的 commit message / PR 标题。
