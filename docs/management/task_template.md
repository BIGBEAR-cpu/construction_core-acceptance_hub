# 任务书模板（Task Template）

## 背景
- 说明业务背景、问题来源、上下游依赖。

## 目标
- 明确本任务要实现的结果与验收期望。

## 允许修改范围
- 允许修改的目录/文件。
- 允许新增的模块与文档。

## 禁止事项
- 不允许修改的目录/文件。
- 不允许执行的高风险操作（如大规模重构、删除历史数据等）。

## 输出文件
- 需新增/修改的文件列表。
- 预期产物说明（代码、文档、脚本、报告等）。

## 测试命令
- 列出必须执行的测试命令。
- 说明测试环境要求。

## 验收标准
- 功能正确性标准。
- 性能/稳定性/合规性标准。
- 文档完整性标准。

## 最终报告格式
- 需提交开发报告与审核报告。
- 报告必须包含：目标达成情况、修改范围、测试结果、风险说明。
