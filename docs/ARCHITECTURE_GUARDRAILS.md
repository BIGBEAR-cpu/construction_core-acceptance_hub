# ARCHITECTURE GUARDRAILS（架构红线）v0.1

以下红线为后续 Codex 实现员、测试员、审核员必须遵守的硬约束：

1. 不允许子项目重复定义冲突的核心模型。
2. 不允许把 mock 能力描述或交付为真实生产能力。
3. 不允许将未实测功能标记为“已完成”。
4. 不允许知识条目无 citation。
5. 不允许规则条目无 version。
6. 不允许 AI 判断无 confidence。
7. 不允许低置信度结果绕过人工复核。
8. 不允许项目级规则污染通用知识库。
9. 不允许跳过 P0.1 / P1.1 验收冻结。
10. 不允许核心功能没有测试。
11. 不允许文档与代码状态不一致。
12. 不允许加载未知来源向量索引。
13. 不允许把系统描述成可替代专业工程师或专家论证。

## 执行建议
- 每个 PR 必须标注：本次变更阶段（P0/P1...）、是否涉及公共模型、是否涉及知识包版本。
- 每个核心输出必须可追溯到 `source_ref + citation + version + confidence`。
- 每次阶段冻结前必须生成验收报告并归档。
